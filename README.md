# README

This is an example app attempting to get JRuby running with ActiveRecord 6 and Postgres. My config works with MRI so it's not the config. 

# Bug #1 with version 60.2 adapter

## Versions:

* Rails 6.0.3.4
* activerecord-jdbcpostgresql-adapter (60.2-java)
* JRuby 9.2.13.0

## Steps to reproduce bug #1:

Note: I'm using `asdf` as my Ruby Version Manager. It uses a .tool-versions file to set the appropriate version. 

* `rails new jruby_activerecord_test` and config database.yml or clone this repo
* cd to this folder and run `ruby -v` Make sure you're using JRuby. The .tool-versions file sets it to `jruby-9.2.13.0` I also tried `jruby-9.2.7.0` and get the same error.
* Add `gem 'activerecord-jdbcpostgresql-adapter', platforms: [:jruby]` to Gemfile. Don't add `gem 'activerecord-jdbc-adapter', :github => 'jruby/activerecord-jdbc-adapter', platforms: [:jruby]`. I'll show that error in a minute.
* `gem install bundler`
* `bundle install` You should get something like `Bundle complete! 18 Gemfile dependencies, 75 gems now installed.`
* `rake db:create` or `bundle exec rake db:create`
